# 演示与验收说明

## 1. 文档用途

本文档用于课程作业验收时快速复现系统演示流程。验收重点是证明三院异构教务系统可以通过 XML 完成共享课程查询、跨院选课写回、退选写回和统计汇总。

## 2. 启动前准备

在仓库根目录执行：

```bash
./scripts/install_demo_deps.sh
```

该命令安装后端和前端演示依赖。默认演示中，A/B/C 学院后端按照 SQL Server、Oracle、MySQL 的表结构加载演示仓储，集成服务器使用 SQLite 状态库，因此不依赖验收现场安装真实 SQL Server、Oracle、MySQL 服务。

## 3. 一键启动

```bash
./scripts/dev_up.sh
```

启动后等待几秒，服务地址如下：

| 服务 | 地址 |
| --- | --- |
| 学院 A 后端 | `http://localhost:8000` |
| 学院 B 后端 | `http://localhost:8001` |
| 学院 C 后端 | `http://localhost:8002` |
| 集成服务器 | `http://localhost:8081` |
| 前端 | `http://localhost:5173` |

启动脚本会重置集成服务器演示库，并重新生成 `data/generated/college_a_demo.sql`、`data/generated/college_b_demo.sql`、`data/generated/college_c_demo.sql`。

## 5. 手工演示脚本

### 5.1 登录和本院课程

1. 打开 `http://localhost:5173/college/A/login`。
2. 使用 `a_student1 / 123456` 登录。
3. 进入本院课程页面，确认能看到 A 学院课程列表。

预期结果：页面显示本院课程，后端返回 XML 格式课程数据。

### 5.2 共享课程查询

1. 切换到共享课程页面。
2. 查看来自 A/B/C 学院的共享课程。
3. 选择一门 B 学院或 C 学院课程，例如 `B001`。

预期结果：共享课程中包含 `collegeId`，用于识别课程来源学院。

### 5.3 跨院选课写回

1. 在共享课程页面点击选课。
2. 系统向集成服务器提交 XML 选课请求。
3. 集成服务器调用目标学院内部写回接口。

预期结果：返回 `ENROLLED` 状态，并生成 `enrollmentId`。

### 5.4 我的选课

1. 打开“我的选课”页面。
2. 查看刚才选择的跨院课程。

预期结果：页面能查询到跨院选课记录，课程编号与选课返回结果一致。

### 5.5 退选流程

1. 在“我的选课”页面点击退选。
2. 集成服务器根据 `enrollmentId` 找到目标学院。
3. 集成服务器调用目标学院退选写回接口。

预期结果：返回 `WITHDRAWN` 状态，退选后“我的选课”不再显示该记录。

### 5.6 统计汇总

1. 打开 `http://localhost:5173/integration`。
2. 查看三院学生、课程和选课统计。

预期结果：三院统计均为学生 50、课程 10、选课 250，总计学生 150、课程 30、选课 750。

## 6. 自动验收

启动服务后执行：

```bash
./scripts/smoke_test.sh
```

自动验收覆盖：

- A/B/C/集成服务器健康检查。
- 三院课程数量均为 10。
- 三院内部统计均为 50/10/250。
- 集成服务器共享课程包含 B 学院课程。
- A 学生跨院选择 B001 后写回成功。
- “我的选课”能查询到新选课。
- 退选后该记录不再出现在“我的选课”中。
- 集成服务器总统计为 150/30/750。

脚本输出 `Smoke test passed.` 表示验收通过。

## 7. 停止服务

```bash
./scripts/dev_down.sh
```
