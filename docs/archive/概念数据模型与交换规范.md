# 概念数据模型与交换规范

## 1. 文档目的

本文档根据学院 A、B、C 的真实数据库结构，以及集成服务器统一 XML Schema 的要求，定义四个系统之间共享的数据模型、字段口径和交换规范。

本文档的目标是：

- 统一三院对学生、课程、选课和账户信息的表达方式。
- 为联调接口、XML 报文、数据导入导出和课程共享提供统一依据。
- 说明三院内部结构各不相同，但对外交换必须标准化。

## 2. 总体原则

- 以各学院现有表结构为基础，抽象出统一的业务概念。
- 三院内部可以保留各自字段名、字段类型和约束方式。
- 集成服务器不保存业务数据库，只定义统一 XML Schema，并作为标准交换口径。
- 所有跨院交换数据均通过 XML 表达，并以 XSLT 或映射规则完成转换。

## 3. 学院 A 数据库结构（SQL Server）

### 3.1 账户表 Account

| 字段   | 类型        | 约束 | 说明          |
| ------ | ----------- | ---- | ------------- |
| 账户名 | varchar(10) | PK   | 登录账号      |
| 密码   | varchar(6)  |      | 登录密码      |
| 权限   | char(4)     |      | 学生 / 管理员 |

### 3.2 学生表 Student

| 字段     | 类型        | 约束 | 说明         |
| -------- | ----------- | ---- | ------------ |
| 学号     | varchar(12) | PK   | 学生唯一标识 |
| 姓名     | varchar(10) |      | 学生姓名     |
| 性别     | varchar(2)  |      | 性别         |
| 院系     | varchar(10) |      | 所属院系     |
| 关联账户 | varchar(10) | FK   | 关联账户表   |

### 3.3 课程表 Course

| 字段     | 类型        | 约束 | 说明            |
| -------- | ----------- | ---- | --------------- |
| 课程编号 | varchar(8)  | PK   | 课程 ID         |
| 课程名称 | varchar(10) |      | 课程名          |
| 学分     | varchar(2)  |      | 学分            |
| 授课老师 | varchar(10) |      | 教师            |
| 授课地点 | varchar(20) |      | 上课地点        |
| 共享     | char(1)     |      | 是否共享（Y/N） |

### 3.4 选课表 Enrollment

| 字段     | 类型        | 约束 | 说明 |
| -------- | ----------- | ---- | ---- |
| 课程编号 | varchar(8)  | UK   | 课程 |
| 学生编号 | varchar(12) | UK   | 学生 |
| 成绩     | varchar(3)  |      | 成绩 |

> 说明：学院 A 的选课表采用 `(课程编号, 学生编号)` 联合唯一约束。

## 6. 集成服务器统一 XML Schema

集成服务器不保存业务数据，只定义统一 XML Schema，用于三院系统之间的标准化交换。

### 6.1 统一课程结构 formatClass.xsd

| 元素     | 类型         | 说明     |
| -------- | ------------ | -------- |
| classes  | 复合         | 课程集合 |
| class    | 复合         | 单门课程 |
| id       | string       | 课程编号 |
| name     | string       | 课程名称 |
| time     | unsignedByte | 课时     |
| score    | unsignedByte | 学分     |
| teacher  | string       | 授课教师 |
| location | string       | 上课地点 |
| collegeId | string       | 开课学院，可选 |

### 6.2 统一学生结构 formatStudent.xsd

| 元素     | 类型   | 说明     |
| -------- | ------ | -------- |
| students | 复合   | 学生集合 |
| student  | 复合   | 单个学生 |
| id       | string | 学号     |
| name     | string | 姓名     |
| major    | string | 专业     |

### 6.3 统一选课结构 formatClassChoice.xsd

| 元素    | 类型         | 说明     |
| ------- | ------------ | -------- |
| choices | 复合         | 选课集合 |
| choice  | 复合         | 单条选课 |
| sid     | string       | 学生学号 |
| cid     | string       | 课程编号 |
| score   | unsignedByte | 成绩     |

## 7. 统一字段映射关系

### 7.1 学生字段映射

| 统一字段 | A 库来源     | B 库来源     | C 库来源    |
| -------- | ------------ | ------------ | ----------- |
| id       | Student.学号 | Student.学号 | Student.Sno |
| name     | Student.姓名 | Student.姓名 | Student.Snm |
| major    | Student.院系 | Student.专业 | Student.Sde |

### 7.2 课程字段映射

| 统一字段 | A 库来源        | B 库来源    | C 库来源   |
| -------- | --------------- | ----------- | ---------- |
| id       | Course.课程编号 | Course.编号 | Course.Cno |
| name     | Course.课程名称 | Course.名称 | Course.Cnm |
| time     | 0 或补充值      | Course.课时 | Course.Ctm |
| score    | Course.学分     | Course.学分 | Course.Cpt |
| teacher  | Course.授课老师 | Course.老师 | Course.Tec |
| location | Course.授课地点 | Course.地点 | Course.Pla |

> 说明：学院 A 的课程表未显式保存课时字段，因此导出到统一 XML 时可按 0 处理，或在导出层补充默认值。

### 7.3 选课字段映射

| 统一字段 | A 库来源            | B 库来源            | C 库来源       |
| -------- | ------------------- | ------------------- | -------------- |
| sid      | Enrollment.学生编号 | Enrollment.学号     | Enrollment.Sno |
| cid      | Enrollment.课程编号 | Enrollment.课程编号 | Enrollment.Cno |
| score    | Enrollment.成绩     | Enrollment.得分     | Enrollment.Grd |

## 8. 交换规则

- 学院内部字段名可以不同，但导出到集成服务器时必须统一转换为标准 XML 结构。
- 集成服务器仅处理 formatClass.xsd、formatStudent.xsd、formatClassChoice.xsd 所定义的标准字段。
- 状态、路由、时间戳等业务元数据如需传递，可放在 API 外层请求封装中，不进入核心数据结构。
- 课程共享时，三院导出的课程数据均应遵循统一课程结构；学生基本信息按统一学生结构导出；选课结果按统一选课结构导出。

## 9. 与 API 协议的关系

本文档与 [API 协议与联调说明](API协议与联调说明.md) 配套使用：

- 本文档定义数据库字段、统一 XML Schema 和字段映射。
- API 协议文档定义接口地址、请求方法、错误码和联调流程。
- 后续示例数据脚本、导入导出逻辑和集成服务器实现都应以本文档为准。

## 10. 建议的使用顺序

1. 先确认三院真实表结构。
2. 再按本文档完成字段映射。
3. 再依据 XML Schema 实现导入导出。
4. 最后按照 API 协议完成联调。
