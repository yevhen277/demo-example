开发与部署注意事项

- 开发环境：本地开发和联调时使用 `localhost`，例如 `http://localhost:8081`。
- 部署环境：测试/生产环境使用内网地址 `http://10.60.254.43:8081`。

以下为逐条注意事项：

- 配置文件：不要将生产凭据或真实连接字符串写死在代码中，使用根目录的 `.env` 或环境变量管理（仓库保留 `.env.example` 作为示例）。
- CORS 与防火墙：本地允许 `localhost`；部署到 `10.60.254.43` 时请在防火墙或反向代理中放行默认端口 `8081`。
- 服务发现：若多个服务部署在同一网段，请在部署配置中写入主机名/端口，避免在代码中硬编码，建议使用环境变量或配置中心。
- 联调说明：开发时可使用 `localhost` 的占位服务（mock）；联调/演示时请切换到 `http://10.60.254.43:8081` 并通知对方放行网络。
- 示例环境变量（`.env.example`）：

```
API_BASE_URL=http://localhost:8081  # 开发时使用，联调/部署时替换为 http://10.60.254.43:8081
```

- 分支管理提醒：修改代码前请新建个人分支并通过 Pull Request 合并，避免直接在 `main` 上修改；不要在分支中提交真实凭据（如 `.env`）。
- 其他建议：保持分支短小并经常同步主分支以减少冲突；合并前做基本功能验证。
